## PLAN deltas discovered by live evidence (consolidated)

Ledger §G items 1–11 stand (metadata truth = information_schema.columns.COLUMN_TYPE;
Decimal256 is a Trino-ceiling not wire limit; LARGEINT is text + getBigDecimal throws on the
negative extreme; getDate/getTimestamp YEAR/zone-shift hazards ⇒ LocalDate/LocalDateTime
reads; DOUBLE Infinity is a READ failure ⇒ getString reads; string-array decode provably
ambiguous ⇒ hard NO-GO; drop the per-scan RSMD probe; DATETIME max precision 6;
brikk FunctionHazard lacks target name + no release; multi-FE unproven; boolean
discrimination via tinyint(1)). New since the ledger:

- **The PLAN §6.2 `array_contains(...) = 1` wrapper defeats the inverted index**: profile
  counters show 0 rows index-filtered wrapped vs 997,997 bare
  (`evidence/inverted-index-explain-p2b.md`). Rendering is the BARE truthy form,
  predicate-equivalent and emitted only as top-level WHERE conjuncts (no composition rules
  for predicate-level rewrites — enforced structurally).
- **Doris `arrays_overlap` matches NULL elements to each other** (`[1,null]×[null,4]` → 1
  where Trino → NULL): unguarded pushdown would over-return; fixed with the
  `array_filter(x -> x IS NOT NULL, left)` wrapper, all shapes re-proven.
- **Array string elements carry NO escaping** (spike correction of the P0 probe report's
  `repr()` artifact): the byte-identical ambiguity pair is the reason ARRAY<VARCHAR> is
  never native.
- **DATETIME array wire fractions are variable-width (1..6 = declared scale)** — the spike's
  fixed `.SSSSSS` parse was wrong for datetime(3); production parses 1–6 digits strictly.
- **P1b query_timeout finding**: Connector/J's client-side `Statement.setQueryTimeout` never
  fires during the streaming drain; server-side `SET query_timeout` kills send-blocked
  queries ("killed by timeout checker") — the session property applies server-side.
- **P3 TopN findings**: Doris 4.1.3 supports native `NULLS FIRST/LAST` (SR K6's ISNULL trick
  unnecessary); `default_order_by_limit = -1` (no bare-ORDER-BY truncation here — and the
  connector never emits bare ORDER BY); `LIMIT 70000 > 65535` exact ⇒ PLAN §9.2 risk closed
  for pushed shapes; Doris NOT/AND/OR three-valued logic is cell-identical to Trino's ⇒
  composition over value-identical operands (array_position comparisons) is safe, encoded as
  a structurally separate value-safe rewriter tier.
- Assembly note (P1a): `mysql-connector-j` is compile-scope (Driver referenced in code),
  matching upstream trino-mysql, vs the assembly-proof's `runtimeOnly` sketch.
